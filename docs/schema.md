# DB Schema

| Table             | Key columns                                                                                                                                                                                                                                                                                           |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `users`           | id, email, passwordHash                                                                                                                                                                                                                                                                               |
| `userSettings`    | userId (PK, FK) — cuisinePrefs[]/dietaryRestrictions[] (defaults new plans inherit), calorieTarget/proteinTarget/carbsTarget/fatTarget (nullable; NULL → global default). 1:1 with users, created on first save                                                                                       |
| `sessions`        | id (PK), userId FK, expiresAt                                                                                                                                                                                                                                                                         |
| `meals`           | id, userId FK (NULL = global/shared, else personal), name, calories, proteinG, carbsG, fatG, tags[], allowedSlots[] (empty = any slot), imageUrl, description, ingredients[] (free text, source of truth for authoring/display), instructions, timeMinutes, difficulty, servings (default 1)          |
| `ingredients`     | id, name (unique, canonical capitalized form)                                                                                                                                                                                                                                                         |
| `mealIngredients` | id, mealId FK (cascade), ingredientId FK, position, qty (nullable numeric), raw (original free-text line). Derived from a meal's `ingredients[]` on every write (see `syncMealIngredients` in `src/lib/server/meals.ts`) so the shopping list can sum real quantities instead of re-parsing free text |
| `plans`           | id, userId FK, name, weekStart, cuisinePrefs[], dietaryRestrictions[]                                                                                                                                                                                                                                 |
| `weekSlots`       | (planId, date, mealType) composite PK, mealId FK                                                                                                                                                                                                                                                      |
| `mealFavorites`   | (userId, mealId) composite PK, both FK cascade — per-user bookmark on a meal                                                                                                                                                                                                                          |
